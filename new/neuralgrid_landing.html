How it works

One API call. Zero GPU decisions.

The routing happens between your request and the result — you never pick hardware.

1

Submit your job

Send a prompt, image, or audio job through a single API call. No tier parameter to fill in.

2

Estimator classifies it

VRAM requirement is calculated from the model, quantization, and input size before anything runs.

3

Cheapest sufficient tier wins

Live prices across providers are compared. The lowest-cost node that can actually complete the job is picked.

4

Job runs, result returns

Dispatched, monitored, retried if needed. You get the output and the receipt — same as before, just cheaper.

Platform

Built entirely on AMD Developer Cloud

No aggregation across marketplaces, no third-party nodes. One fleet, sized correctly per job.

AMD Instinct MI300X
192GBHBM3 per node

A single MI300X carries enough VRAM to run 70B+ models on one node — and enough headroom to partition into smaller instances for lighter jobs. NeuralGrid's scheduler packs jobs onto the smallest AMD Instinct partition that fits, across MI210 and MI300X capacity, entirely within AMD Developer Cloud.
